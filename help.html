このアプリについて

WDR工数ビューアは、WDR（Web Daily Report）からダウンロードした工数テキストを読み込み、集計・グラフ表示・AIによる総括コメント生成を行う Web アプリです。Excel で手作業していた集計や俯瞰を、ファイルを選ぶだけで素早く行えます。

アップロードしたファイルの内容をもとに、次の2種類の分析モードを自動判定します。

モード 用途 主な集計
製番別 1案件（製番）の工数分析 拠点・スタッフ別 / 作業種別 / 月別
部門・グループ別 部門・グループの稼働分析 個人別・稼働率 / クライアント別 / 製番別 / 事業所別 / 月別 など

かんたんな使い方

  1. 1 WDR から工数テキスト（TXT / TSV / CSV）をダウンロードします。
  2. 2 アプリのファイル選択欄でダウンロードしたファイルを選びます（ドラッグ＆ドロップも可）。
  3. 3 集計する をクリックします。
  4. 4 表示されたタブを切り替えて、グラフと一覧で工数の傾向を確認します。
  5. 5 （任意）一覧の行をクリックすると明細が表示されます。（任意）AI分析で総括コメントを生成できます。
ヒント：別のファイルを分析したいときは、画面上部の テキストデータを読み込む から最初の画面に戻れます。

データの準備（WDR からのダウンロード）

マネージャーの場合

A 製番別データ

  1. 管理メニューの「検索」で製番を検索
  2. 検索結果の製番をクリックして詳細モーダルを開く
  3. 「ダウンロード」でテキストを保存

B 部門・グループ別データ

  1. 管理メニューで対象の部門またはグループを選択
  2. 分析したい期間を設定
  3. 「ダウンロード」でテキストを保存

マネージャー以外の場合

製番別データ（A）のみ取得できます。

  1. 登録済み日報の製番をクリックして詳細モーダルを開く
  2. 「ダウンロード」でテキストを保存

対応ファイル形式

  • 拡張子：TXT / TSV / CSV
  • 文字コード：UTF-8 または Shift_JIS（自動判定）
  • 区切り：タブ区切り（TSV）を想定（WDR の標準エクスポート形式）

ファイルの読み込みと集計

ファイル選択欄をクリックするか、ファイルをドラッグ＆ドロップして選択します。選択後、ファイル名が表示されます。

集計する を押すと読み込みと集計が始まります。ヘッダー行の列名から列の意味を自動判定するため、WDR からそのままダウンロードしたファイルを使えます。

集計が完了すると、ファイルの種類に応じたモード（製番別 / 部門・グループ別）の結果が表示されます。画面上部のバッジでも現在のモードを確認できます。

製番別モード

1つの製番（案件）に関する工数を分析します。

拠点・スタッフ別

事業所とスタッフ名の組み合わせごとの合計工数を表示します。誰がどれだけ関わったかを把握できます。

作業種別

デザイン・入稿・校正など、作業種別ごとの工数配分を表示します。

月別

データ期間内の各月の合計工数を表示します。繁忙期・閑散期の把握に使えます。

部門・グループ別モード

部門やグループ単位の稼働状況を多角的に分析します。

個人別・稼働率

各メンバーの稼働時間・総時間・稼働率を表示。グラフでは売上「有」または特定「有」の稼働分（青紫）と、TP/TD オプション ON 時の追加稼働分（琥珀色）を色分けします。

クライアント別

クライアントごとの合計工数と担当人数。複数人が関与する場合は作業者別の積み上げグラフになります。

製番別

製番ごとの工数・担当人数・主なクライアント・品名を表示。製番が空の行は「製番なし」に集約されます。製番なしを含めるチェック（デフォルト OFF）で表示の有無を切り替えられます。

事業所別

製番の先頭2桁などから事業所を判定し、事業所ごとに工数を集計します。判定ルールは事業所別の判定を参照してください。

月別

月ごとの合計工数（棒グラフ）と平均稼働率（折れ線）を同時に表示します。

作業者別・クライアント / 作業者別・作業種別

スタッフごとにクライアント構成または作業種別構成を円グラフで表示します。凡例に時間と割合（%）が表示されます。

月別列について：分析期間が2か月以上の場合、個人別・稼働率タブには各月の稼働率（%）列が、クライアント別タブには各月の合計時間（h）列が追加されます。

詳細モーダル

集計一覧の行をクリックすると、その行に含まれる明細データがモーダルで開きます。

  • 列ヘッダーをクリックすると、その列で昇順・降順に並び替えできます
  • 作業内容・品名など長いテキストは2行で省略表示され、マウスを乗せると全文を確認できます
  • 部門モードでは売上・特定の有無がバッジで表示されます
  • モーダル上部に件数・合計時間（稼働率タブでは稼働率も）が表示されます

分析期間とオプション

分析期間フィルタ（部門モード）

部門・グループ別モードでは、画面上部の分析期間で日付範囲を指定し、適用を押すとその期間内のデータだけで再集計します。デフォルトはファイルに含まれるデータ全体の期間です。

TP/TD製番の売上無・特定無も稼働に含める

分析期間の右側にあるチェックボックスです。ON にすると、製番が TP / TD で始まり、売上・特定が両方「無」の行も稼働時間に含めます。個人別・稼働率のグラフでは琥珀色で積み上げ表示されます。

製番なしを含める（製番別タブ）

製番別タブ内のチェックボックスです。デフォルトでは「製番なし」の行をグラフ・一覧から除外します。製番なしのボリュームが大きい場合でも、個別製番の比較がしやすくなります。必要なときだけ ON にしてください。

稼働率の計算式

稼働時間 = 売上「有」または特定「有」の行の作業時間の合計
         （TP/TD オプション ON 時は該当行も加算）
総時間 = 同一作業者の全行の作業時間の合計（期間内）
稼働率 = 稼働時間 ÷ 総時間 × 100

キーワード絞り込み

各タブの検索欄にキーワードを入力すると、一覧とグラフがリアルタイムで絞り込まれます（製番・クライアント名・作業者名など）。

AI分析（任意機能）

集計結果のサマリー（明細行は含みません）を LLM に送信し、総括コメントを生成します。データの傾向把握や報告書作成のたたき台としてご利用ください。

使い方

  • 集計前：AI分析も実行する にチェック → 集計後に自動でデフォルト分析を実行
  • 集計後：3つの分析ボタンから切り口を選んで実行
モード 分析ボタン 自動実行（チェック ON 時）
製番別 業務サマリー / 偏り・集中度 / 気になる点 業務サマリー
部門別 チーム総括 / 個人・稼働率 / リスク・依存 チーム総括

AI 設定（管理者向け）

ヘッダー右上の ⚙ ボタンは管理者のみ表示されます。GB10 / Gemini / LM Studio の切り替えや API 設定を行えます。設定はサーバーに保存され、全利用者に共通で適用されます。

よくある質問

データ形式を判定できないと表示される
ヘッダー行に「日付」「作業時間」「作業内容（または業務内容）」「担当者名（スタッフ/社員名など）」が含まれているか確認してください。WDR からそのままダウンロードしたファイルをお使いください。
部門モードで明細が空になる
ヘッダー行の直後にデータ行があるか確認してください。# で始まるメタ情報行の後にデータが続く形式が一般的です。
稼働率が想定と違う
売上・特定の「有/無」列の値、分析期間の設定、TP/TD オプションの ON/OFF を確認してください。期間を変更した場合はその期間内の行だけで再計算されます。
製番別グラフで棒が小さくて見づらい
製番別タブの「製番なしを含める」のチェックを OFF にしてください（デフォルト）。製番なしを除外すると個別製番の比較がしやすくなります。
画面のレイアウトが崩れる
社内ネットワークから Tailwind CSS の CDN にアクセスできるか確認してください。更新後はブラウザのハードリロード（Ctrl+Shift+R）をお試しください。
AI分析が失敗する
管理者に AI 設定（プロバイダ・API キー・URL）を確認してください。GB10 利用時はネットワーク到達性や Mixed Content の制約にも注意が必要です。

用語集

製番
製造番号。案件を識別するコードです。
稼働率
売上または特定が「有」の時間が、総作業時間に占める割合（部門モード）。
売上 / 特定
WDR の工数行に付くフラグ。「有」なら稼働（売上計上または特定作業）として扱います。
TP/TD 製番
製番が TP または TD で始まる案件。オプションで非稼働行も稼働に含められます。
製番なし
製番欄が空、または「なし」「無し」などと記載された行の集約ラベルです。
GB10
NVIDIA の Grace Blackwell 世代に属する AI 向け統合チップです。CPU と GPU を高帯域で統合した構成により、生成 AI、LLM 推論などをローカル環境で効率よく実行できます。 ITDC で GB10 を購入し、vLLM を導入して Qwen3.6 を API として運用しています。これにより、クラウド依存を抑えつつ、社内での AI 推論基盤を手元の環境で安定運用できる状態にしています。 本アプリの AI 分析では、この社内 API（GB10 プロバイダ）をデフォルトとして利用できます。
WDR
Web Daily Report。日報・工数の登録・管理システムです。
