# WDR工数ビューア

**Version 2.2.15**

WDR（Web Daily Report）からダウンロードした工数テキスト（TSV/CSV）をアップロードし、集計・可視化する Web アプリです。  
オプションで LLM による総括・インサイトを生成できます。

アップロードしたファイル形式を **自動判定** し、次の2種類の分析を切り替えます。

| モード | 用途 | 主な集計 |
|--------|------|----------|
| **製番別** | 1案件（製番）の工数分析 | 拠点・スタッフ別 / 作業種別 / 月別 |
| **部門・グループ別** | 部門・グループの稼働分析 | 個人別・稼働率 / クライアント別 / 製番別 / 事業所別 / 月別 / 作業者別・クライアント / 作業者別・作業種別 |

**Google Apps Script（Clasp）** または **社内オンプレミス Web サーバー** に配置して利用できます。  
単一 HTML ファイル（`index.html`）＋ GAS プロキシ（`main.gs`）構成です。

## 主な機能

### 共通

- **ファイル取り込み** — TXT / TSV / CSV（Shift_JIS・UTF-8 自動判定）
- **形式自動判定** — 製番別 / 部門・グループ別をファイル内容から判別
- **詳細モーダル** — 集計行クリックで明細一覧（列ヘッダーでソート）
- **グラフ表示** — 各タブに棒グラフ（Chart.js）
- **AI 分析（任意）** — 集計サマリーを LLM に渡し、総括コメントを生成
- **AI プロバイダ切替（⚙・管理者のみ）** — GB10 / Gemini / LM Studio。設定は GAS サーバーに保存され全員共通

### 製番別集計

- 拠点・スタッフ別 / 作業種別 / 月別の 3 タブ
- 月別はデータ期間の全月をグラフ・一覧に表示
- PM 向け AI 総括（部門名を参照した作業内容の解釈）

### 部門・グループ別集計（v2.0.0 新規）

- **個人別・稼働率** — 売上「有」または特定「有」の時間 ÷ 総作業時間
- **クライアント別** — 工数・担当人数
- **製番別** — 工数・担当人数（製番なしは「製番なし」に集約）。**製番なしを含める**チェックで表示切替（デフォルトは除外）
- **事業所別** — 製番先頭2桁・品名【】・クライアント名から事業所を判定して集計
- **月別** — 合計工数（棒）と平均稼働率（折れ線）の複合グラフ
- **分析期間フィルタ** — 日付範囲を指定して再集計（デフォルトはデータ全体）
- **TP/TD製番オプション** — 売上無・特定無の TP/TD 製番を稼働に含めるか選択可能
- **グラフの色分け** — 稼働率グラフで TP/TD 分を琥珀色で積み上げ表示、製番別グラフでも TP/TD 製番を識別色で表示
- **インクリメンタルサーチ** — 各タブでキーワード絞り込み
- **月別列（2か月以上の分析期間）** — 個人別は月ごとの稼働率、クライアント別は月ごとの合計時間
- **詳細モーダル強化** — 品名表示、売上/特定バッジ、長文は「…」＋ホバーで全文
- **担当者別グラフ** — クライアント別・製番別・事業所別で複数人関与時は作業者ごとの工数割合を積み上げ表示
- **作業者別・クライアント** — スタッフごとにクライアント構成を円グラフで表示
- **作業者別・作業種別** — スタッフごとに作業種別の構成を円グラフで表示
- マネージャー向け AI 総括（稼働率・クライアント集中度・製番分散）

## 前提条件

- **Clasp 利用時**: Node.js、`clasp` CLI、Google アカウントでのログイン
- **オンプレ利用時**: 社内 Web サーバー（IIS / nginx / Apache 等）
- 利用 PC が AI API に到達できること
- AI 分析を使う場合
  - **GB10（本番想定）**: 閉域網内の OpenAI 互換 API
  - **Gemini**: インターネット経由で Google AI API に到達可能であること
  - **LM Studio**: 利用 PC 上で LM Studio サーバーが起動していること

## セットアップ

### Clasp でデプロイ（GAS）

```bash
git clone https://github.com/k-suehiro/job-analytics.git
cd job-analytics
clasp login
clasp push
```

Web アプリとしてデプロイする場合は GAS エディタから「デプロイ」→「新しいデプロイ」を実行してください。

**AI 設定（管理者）**

- ヘッダー右上の ⚙ ボタンは **GAS スクリプト所有者**（および `AI_SETTINGS_ADMIN_EMAILS` に登録したメール）のみ表示されます
- 保存した設定は GAS の Script Properties に格納され、**全利用者に共通**で適用されます
- Web アプリは **「自分として実行」** のままで問題ありません。利用者は **Google アカウントでログイン** した状態で開いてください
- メールアドレスが取得できない場合は、管理者用 URL（`?ak=...` 付き）で開くと ⚙ が表示されます（下記）

**管理者用 URL の取得（設定ボタンが出ない場合）**

1. GAS エディタで `logAiSettingsAdminAccessUrl` を実行
2. 実行ログに表示された URL（`?ak=...` 付き）をブックマーク
3. その URL から開くと ⚙ が表示されます（他の人には通常 URL を共有してください）

任意: `registerAiSettingsAdminEmails` を GAS エディタで実行すると、自分のメールを管理者に明示登録できます。

### オンプレミス Web サーバー

`index.html` を公開ディレクトリに配置します。

| サーバー | 配置例 |
|---------|--------|
| IIS | `C:\inetpub\wwwroot\job-analytics\index.html` |
| nginx | `/var/www/html/job-analytics/index.html` |

GB10 利用時は **HTML を HTTP で配信** することを推奨します（Mixed Content 回避）。

## 使い方

エンドユーザー向けの詳細ガイドは **[help.html](help.html)**（アプリ画面上部の「ヘルプ」リンクからも開けます）を参照してください。

### 基本フロー

1. WDR から工数テキストをダウンロード（製番別 or 部門・グループ別）
2. ファイルを選択して **集計する** をクリック
3. 形式に応じた集計結果が表示される
4. （任意）**AI分析も実行する** にチェック、または集計後に分析ボタン（3種類）を実行

集計後の AI 分析は横並びの3ボタンから切り口を選べます。

| モード | ボタン |
|--------|--------|
| 製番別 | 業務サマリー / 偏り・集中度 / 気になる点 |
| 部門別 | チーム総括 / 個人・稼働率 / リスク・依存 |

「AI分析も実行する」チェック時は、製番別なら **業務サマリー**、部門別なら **チーム総括** が自動実行されます。

### データの取得方法

画面上部の説明を参照してください。

- **マネージャー** — WDR 管理メニューから取得（A: 製番別 / B: 部門・グループ別を横並びのカードで案内）
- **マネージャー以外** — 製番別（A）のみ（登録済み日報の製番詳細からダウンロード、カード表示）

### 一覧の月別列

分析期間が **2か月以上** の場合のみ、次のタブに月別列が追加されます（1か月のみのときは非表示）。

| タブ | 月別列の内容 |
|------|-------------|
| 個人別・稼働率 | 各月の稼働率（%） |
| クライアント別 | 各月の合計時間（h） |

### 部門モードの稼働率

```
稼働時間 = 売上「有」または特定「有」の行の作業時間の合計
         （オプション ON 時は TP/TD 製番の売上無・特定無も加算）
総時間   = 同一作業者の全行の作業時間の合計（期間内）
稼働率   = 稼働時間 ÷ 総時間 × 100
```

**TP/TD製番の売上無・特定無も稼働に含める**（分析期間の右側）を ON にすると、製番が `TP` / `TD` で始まり売上・特定が両方「無」の行も稼働時間に含めます。

**グラフの見方（個人別・稼働率）**

| 色 | 意味 |
|----|------|
| indigo（青紫） | 売上「有」または特定「有」の稼働分 |
| 琥珀色 | TP/TD 製番の売上無・特定無（オプション ON 時のみ積み上げ） |

製番別タブのグラフでは、TP/TD で始まる製番の棒を琥珀色で表示します。クライアント別・製番別・事業所別で **2名以上** が関与する場合は、作業者ごとの工数割合を積み上げ棒グラフで表示し、ホバーで名前と時間・割合を確認できます。担当者は原則すべて個別表示し、「その他」は **16名以上** いて末尾に **2名以上** まとまる場合のみ使います。

**製番別タブの「製番なしを含める」**

デフォルトでは「製番なし」行を集計・グラフから除外します。製番なしのボリュームが大きい場合でも、個別製番の比較がしやすくなります。チェックを ON にすると従来どおり製番なしも含めます。

**事業所別の判定ルール**

製番の先頭2桁（大文字小文字は区別しない）などから事業所を判定します。

| 先頭2桁 | 事業所 |
|--------|--------|
| TK, FK | 東京事業所 |
| HM | 浜松事業所 |
| NA | 名古屋事業所 |
| MT | 松本事業所 |
| OS | 大阪事業所 |
| SL | 国内支援室CSC |
| MP | クライアントに「シープラス」または「大野印刷」→ **シープラス**、それ以外 → **海外事業部** |
| TP, TD | プロダクションセンター |
| EK | 品名の【…】から判定（下記） |
| HP | 国内支援室浜松プリントセンター |
| 製番なし | 製番なし |
| 上記以外 | その他 |

EK 製番の品名【】判定: 【本社】【人事総務課】→ 人事総務部、【財務経理部】→ 財務経理部、【役員室】→ 役員室、【】なし → 人事総務部（本社扱い）。その他の【…】は括弧内の文字を事業所名として使用します。

分析期間を変更した場合、分子・分母ともに **その期間内の行のみ** で再計算されます。

## 入力データ形式

WDR からエクスポートしたタブ区切り（TSV）テキストを読み込みます。**ヘッダー行の列名**から列位置を自動判定し、下記の列名は互いに同義として扱います。

| 意味 | 読み取る列名（いずれか） |
|------|-------------------------|
| 日付 | 日付 |
| 拠点 | 事業所 / 所属 |
| 部署 | 部署 |
| 担当者 | スタッフ / 作業者名 / 社員名 |
| 作業種別 | 作業種別 |
| 作業内容 | 作業内容 / 業務内容 |
| 作業時間 | 作業時間 |
| 社員番号 | 社員番号 |
| 製番 | 製番 / 製造番号 |
| クライアント | クライアント / 得意先 |
| 品名 | 品名 |
| 売上・特定 | 売上 / 特定（有/無） |
| 役務単価 | 役務単価 |

次の列は **読み込みません**（あっても無視）: クレーム、売上完了、親カテゴリ、子カテゴリ、FMコード、Unique ID、作業コード、担当営業

### 製番別

```
#日付	事業所	部署	スタッフ	作業種別	役務単価	作業内容	作業時間
```

ヘッダーに **作業種別** と **事業所/所属** があり、売上・製番など部門向け列がない場合に製番別として集計します。

### 部門・グループ別

```
日付	社員番号	作業者名	製番	売上	特定	クライアント	品名	...	作業内容	作業時間
```

または得意先形式:

```
得意先	品名	製造番号	担当営業	日付	所属	社員名	作業コード	業務内容	役務単価	作業時間	クレーム	売上	売上完了
```

ヘッダーに **売上・特定・製番/製造番号・クライアント/得意先・社員番号** のいずれかがある場合、部門・グループ別として集計します（特定列がなくても可）。

メタ情報の例:

```
#プロダクションセンター浜松2 / 浜松2制作2G
#期間:2025-07-01-2026-06-30
#製番: 無し含む
```

### 共通

- 文字コード: UTF-8 / Shift_JIS（CP932）を自動判定
- `#` で始まる行はメタデータまたはコメントとして処理

## AI 分析について

LLM には **明細行ではなく集計サマリー** のみ送信します。

| モード | 送信内容の例 |
|--------|-------------|
| 製番別 | スタッフ別上位 20 件、作業種別上位 15 件、月別全件 |
| 部門別 | 個人稼働率、クライアント別、製番別（製番なしの含否反映）、事業所別、月別のサマリー |

## 開発メモ

### 更新の反映

- **Clasp**: `clasp push` 後、GAS の Web アプリを再デプロイ（または最新版を利用）

## トラブルシューティング

| 症状 | 確認ポイント |
|------|-------------|
| データ形式を判定できない | ヘッダー行に日付・作業時間・作業内容/業務内容・担当者名があるか確認 |
| 部門モードで明細が空 | ヘッダー行の次からデータ行があるか確認 |
| 稼働率が想定と違う | 売上・特定の「有/無」列、分析期間、TP/TD オプションの ON/OFF |
| 設定ボタン（⚙）が表示されない | 管理者用 URL（`?ak=...` 付き）で開く。GAS エディタで `logAiSettingsAdminAccessUrl` を実行して URL を取得 |
| GB10 で Mixed Content | GAS（HTTPS）から社内 HTTP API へは直接接続不可。GAS プロキシまたは HTTP 配信を検討 |
| 画面が崩れる | Tailwind CDN に到達できているか |
